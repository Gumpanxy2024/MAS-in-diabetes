# 方案 A · Pencil 变量表 + 页面区块配色（一页）

> **用途**：在 Pencil（`.pen`）里画原型时统一用色；**信息架构、区块、适老化尺寸、间距与圆角**仍以 [`FRONTEND_PLAN.md`](./FRONTEND_PLAN.md) 为准。  
> **说明**：文档 §1.2 暖色色板由本页的「方案 A」替代；§1.1 / §1.3 / §1.4（含患者底部 Tab 60px、医生左侧 Sidebar、字号与 8px 间距基数）**不变**。

---

## 1. Pencil 变量（与当前 `.pen` 键名对齐）

在 `.pen` 的 `variables` 中沿用下列 **键名**（节点里写 `$bg`、`$primary` 等），仅把 **取值** 更新为方案 A：

| Pencil 键名 | 方案 A 取值 | 语义 |
|-------------|-------------|------|
| `bg` | `#F6F7FB` | 页面画布（轻盈冷灰底，非黄奶油） |
| `card` | `#FFFFFF` | 卡片 / 表单面板表面 |
| `border` | `#E5E7EB` | 描边、分割线、输入框边框 |
| `text1` | `#111827` | 主标题 / 正文 |
| `text2` | `#4B5563` | 次要说明、表头 |
| `text3` | `#9CA3AF` | 占位、辅助元信息 |
| `primary` | `#2563EB` | 主按钮、选中 Tab、关键链接、主数据强调 |
| `primary-light` | `#BFDBFE` | 轻量高亮条、图标点缀（低面积） |
| `primary-dark` | `#1D4ED8` | 主按钮 Hover / 按下 |
| `nav-bg` | `#0B1220` | 医生 Sidebar；患者端 **若** 用顶栏/底栏深色条时同用（替代 §1.4 木棕 `#3E2723`，布局仍为左侧栏 / 底栏） |
| `nav-fg` | `#F9FAFB` | 导航上文字与图标 |
| `success` | `#16A34A` | 绿码 / 成功 |
| `warning` | `#F59E0B` | 黄码 / 警示（**小面积**：徽章、标签，不当整页底色） |
| `danger` | `#DC2626` | 红码 / 错误 / 删除 |
| `info` | `#0284C7` | 提示条、信息图标（与 `primary` 分工：偏说明，非主行动） |
| `surface2` | `#F3F4F8` | 表头底、列表行 Hover、浅分区（可选） |
| `primarySoft` | `#EFF6FF` | 选中 Tab 弱底、信息条弱底（可选） |
| `record` | `#E11D48` | **录音按住**主钮（与 `danger` 语义分离时优先用） |

**录音主按钮（§1.2 原「醒目红」）**：优先 **`$record`**；若变量未建，可用 **`$danger`** 代替。

**卡片阴影（§1.3）**：仍可用 `0 2px 8px rgba(17,24,39,0.08)`（把原棕 alpha 改为冷灰蓝 alpha，更贴方案 A）。

## 2. 描边写法（Pencil 节点）

带边框的 `frame` 使用对象形式（勿用扁平 `strokeWidth`）：

```json
"stroke": { "fill": "$border", "thickness": 1 }
```

## 4. 与 FRONTEND_PLAN 的对照备忘

| FRONTEND_PLAN 条目 | 方案 A 下的落实 |
|---------------------|----------------|
